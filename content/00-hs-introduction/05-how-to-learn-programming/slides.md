<section>
	<h4>The Developmental Epistemology of Computer Programming</h4>
	<p>Lister & Teague, et. al.</p>
	<p><i>AKA how we learn to program</i></p>
</section>
<section>
	<h3>1. Sensorimotor</h3>
	<ul style="font-size: 3.5rem">
		<li>Cannot trace code with >= 50% accuracy</li>
		<li>Dominant problem-solving strategy is trial and error</li>
	</ul>
</section>
<section>
	<h3>2. Preoperational</h3>
	<ul style="font-size: 3.5rem">
		<li>Can trace code with >= 50% accuracy</li>
		<li>Traces without abstracting any meaning from code</li>
		<li>Cannot see relationships between lines of code</li>
		<li>Struggles to make effective use of diagrammatic abstractions of code</li>
		<li>Dominant problem-solving strategy is quasi-random code changes and copious trial runs</li>
	</ul>
</section>
<section>
	<h3>3. Concrete Operational</h3>
	<ul style="font-size: 3.5rem">
		<li>Dominant problem-solving strategy is hasty design, futile patching</li>
		<li>Can establish purpose of code by working backwards from execution results</li>
		<li>Tends to reduce levels of abstraction to make concepts more understandable</li>
	</ul>
</section>
<section>
	<h3>4. Formal Operational</h3>
	<ul style="font-size: 3.5rem">
		<li>Uses hypothetico-deductive reasoning (formulates and tests hypotheses)</li>
		<li>Reads code rather than traces to deduce purpose</li>
	</ul>
</section>

<section>
	<h3>Summary</h3>
	<ul style="font-size: 4rem">
		<li>You can learn to program, just like many other students have before you</li>
		<li>It will take hard work and can be frustrating at times</li>
		<li>You'll need to read and write <b>A LOT</b> of code</li>
		<li>There is no shortcut - you must <b>DO THE WORK</b></li>
	</ul>
</section>
<section>
	<h3>How to Help</h3>
	<ul style="font-size: 4rem">
		<li>Get good, regular sleep</li>
		<li>Exercise regularly</li>
		<li>Engage with the content in the course</li>
		<li>Take notes, draw diagrams, annotate code</li>
		<li>Don't multitask while studying</li>
		<li>Don't copy/paste code - retype it!</li>
		<li>Read code by other developers and try to understand it - use GitHub</li>
	</ul>
</section>
